# Security Policy

## Supported Versions

This package is pre-1.0 and under active development. Security fixes are only provided for the latest released version on the `main` branch — there are no older major versions receiving backports at this time.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please use one of the following private channels:

1. **[Report a vulnerability](../../security/advisories/new)** via GitHub's private vulnerability reporting (Security tab → "Report a vulnerability"). This is the preferred method.
2. Email **creightonfrance@gmail.com** with details if you're unable to use GitHub's private reporting.

Please include as much of the following as you can:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, including a minimal reproduction case if possible (e.g. a malicious notebook or parameter payload, since notebook execution is a security-sensitive part of this package).
- The version of the package (and PHP/Laravel versions) you tested against.

## What to expect

- We'll acknowledge your report as soon as possible.
- We'll keep you informed as we investigate and work on a fix.
- We'll credit you in the release notes when the fix ships, unless you'd prefer to remain anonymous.

Please give us reasonable time to address the issue before any public disclosure.
